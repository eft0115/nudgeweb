# Webnudge Logo Placement Plan

> `material`의 Webnudge 캐릭터 로고를 브랜드 신호, 행동 유도, 섹션 전환 장치로 웹페이지 곳곳에 배치하는 계획.

## 배경 및 목적

- 현재 페이지는 `Nudge Studio` 텍스트와 MUI 아이콘 중심으로 구성되어 있어, Webnudge 캐릭터의 고유한 브랜드 인상이 첫 화면에서 충분히 드러나지 않는다.
- `material/logo_webnudge.png`와 `material/total_webnudge.png`는 땅콩 캐릭터, 손가락 방향 제스처, 표정/포즈 시스템, 앱 아이콘 스타일, CTA 버튼 예시, 브랜드 컬러 시스템을 포함한다.
- 로고는 단순 장식이 아니라 사용자의 시선 흐름을 안내하는 "nudge guide" 역할로 사용한다.
- 기대 효과:
  - 첫 방문 10초 안에 기억되는 브랜드 신호 강화
  - CTA, 서비스, 포트폴리오, 문의 흐름에서 다음 행동 유도
  - 페이지 전반의 톤을 더 친근하고 명확한 Webnudge 브랜드로 통일

## 로고 이미지 학습 요약

| 항목 | 관찰 내용 | 적용 원칙 |
|---|---|---|
| 캐릭터 형태 | 둥근 땅콩 실루엣, 두꺼운 다크 브라운 외곽선, 밝은 베이지 채움 | 작은 크기에서도 외곽선과 얼굴이 보이도록 최소 표시 크기 확보 |
| 기본 제스처 | 오른쪽을 가리키는 손가락 포즈 | CTA, 다음 단계, 문의 버튼 근처에 우선 배치 |
| 방향 변형 | 오른쪽, 아래쪽, 위쪽 제스처 세트 | 섹션 이동 방향에 맞춰 포즈 선택 |
| 표정/상태 | 기본, 웃음, 윙크, 신남, 생각 중, 당황, 슬픔 등 | 섹션의 메시지 감정과 매칭하되 남용 금지 |
| 앱 아이콘 스타일 | 베이지, 피넛 브라운, 화이트, 다크모드 배경 | favicon, 헤더, 모바일 아이콘, 다크 영역용 변형 필요 |
| 브랜드 컬러 | Peanut Beige `#E7C48E`, Peanut Brown `#DB827A`, Dark Brown `#3A2415`, Soft Cream `#FBF4EC`, White `#FFFFFF` | 기존 `background.playful`과 `primary` 토큰 조정 후보 |
| 텍스트 조합 | 위/아래 배치 로고 영역 존재 | 헤더와 푸터에서 심볼+워드마크 조합 사용 |

## 핵심 기능

| # | 기능 | 설명 | 우선순위 |
|---|---|---|---|
| 1 | 헤더 브랜드 로고 교체 | 현재 `AutoAwesomeIcon` 심볼을 Webnudge 심볼 또는 심볼+워드마크로 교체 | 필수 |
| 2 | 히어로 캐릭터 배치 | 첫 화면 오른쪽 또는 하단 가장자리에 기본 오른쪽 포즈 캐릭터를 크게 배치 | 필수 |
| 3 | CTA 버튼 내 미니 캐릭터 | 주요 CTA 버튼 왼쪽에 작은 캐릭터 아이콘을 넣고 화살표는 오른쪽 유지 | 필수 |
| 4 | 섹션별 가이드 포즈 | Problem, Nudge Story, Solution, Contact에 방향/감정별 포즈를 배치 | 필수 |
| 5 | 포트폴리오 카드 브랜드 마커 | 각 프로젝트 카드에 작은 스티커형 캐릭터 또는 결과 배지를 추가 | 선택 |
| 6 | 로딩/상태 표현 | 검색 결과 없음, 문의 완료 등 상태 메시지에 표정 변형 사용 | 선택 |
| 7 | favicon 및 앱 아이콘 | `total_webnudge.png`의 favicon 스타일을 기준으로 웹 아이콘 세트 제작 | 선택 |

## 페이지 배치 계획

| 위치 | 사용할 로고/포즈 | 크기 가이드 | 역할 |
|---|---|---|---|
| Sticky Header | 심볼+`web nudge` 또는 심볼만 | 34-44px | 첫 화면부터 브랜드 인지 고정 |
| Hero Section | 기본 오른쪽 포즈 대형 이미지 | Desktop 280-360px, Mobile 160-220px | 핵심 문장과 CTA를 가리키는 시선 유도 |
| Hero CTA | 미니 심볼 또는 손가락 포즈 | 24-32px | `넛지 체험하기` 행동 강조 |
| Problem Section | 당황/생각 중 포즈 | 72-120px | 사용자의 문제 상황을 시각화 |
| Nudge Story Section | Before/After 단계별 표정 | 80-140px | 상호작용 단계 변화 체감 |
| Solution Section | 위쪽/오른쪽 포즈 | 56-96px | 해결 단계의 진행 방향 안내 |
| Services Section | 작은 기본 캐릭터 | 40-64px | 서비스 카드의 브랜드 마커 |
| Portfolio Section | 스티커 스타일 캐릭터 | 48-72px | 사례 카드의 결과/넛지 포인트 강조 |
| FAQ Section | 책 보기 또는 알아보기 포즈 | 72-96px | 질문 탐색 맥락 강화 |
| Contact Section | 오른쪽 또는 전화하기 포즈 | 120-180px | 문의 폼으로 시선을 유도 |
| Footer | 단색 로고 또는 다크모드 반전 로고 | 36-56px | 브랜드 마무리와 재인지 |

## 대상 사용자

- 주요 사용자: 웹 에이전시 서비스를 검토하는 잠재 고객
- 보조 사용자: 포트폴리오/브랜드 역량을 빠르게 확인하는 채용 담당자 또는 협업 파트너

## 기술적 범위

- 포함:
  - 기존 `NudgeAgencyPage`의 헤더, 히어로, CTA, 섹션 카드, 문의 영역에 로고 이미지 배치 계획 수립
  - `material/logo_webnudge.png`와 `material/total_webnudge.png`를 기준으로 필요한 파생 에셋 정의
  - MUI `Box`, `Stack`, `Button`, `Chip` 구조에 맞는 적용 방식 정의
- 제외:
  - 이번 문서 단계에서는 실제 컴포넌트 수정 및 이미지 추출 작업 제외
  - 캐릭터 애니메이션 직접 구현 제외
  - 외부 이미지 URL 사용 제외
- 제약사항:
  - 현재 개별 포즈 이미지 파일이 분리되어 있지 않으므로, 구현 전 포즈별 PNG/SVG 에셋 추출이 필요하다.
  - 이미지가 흰 배경을 포함할 수 있으므로 투명 배경 에셋 여부를 확인해야 한다.
  - 컴포넌트 수정 시 `/component-work` 워크플로우를 따라야 한다.

## 성공 기준

- 헤더와 히어로에서 Webnudge 캐릭터가 첫 화면의 명확한 브랜드 신호로 보인다.
- 로고/캐릭터가 텍스트 가독성이나 CTA 클릭 영역을 방해하지 않는다.
- 캐릭터 사용 위치가 CTA, 단계 이동, 상태 안내와 연결되어 기능적 의미를 가진다.
- 모바일에서 캐릭터가 과도하게 크게 보이거나 레이아웃을 밀어내지 않는다.
- 브랜드 컬러가 기존 페이지의 파란 primary 중심 톤을 Webnudge 컬러 시스템과 조화되게 전환한다.

## 구현 전 준비 항목

- `logo_webnudge.png`에서 투명 배경 여부 확인 및 필요 시 투명 PNG로 정리
- `total_webnudge.png`에서 다음 에셋 분리:
  - 기본 오른쪽 포즈
  - 아래쪽 포즈
  - 위쪽 포즈
  - 생각 중/당황/웃음/책 보기/전화하기 포즈
  - favicon/app icon 버전
- `src/assets/brand/` 또는 프로젝트 내 기존 에셋 위치를 정하고 파일명 규칙 수립
- theme 후보 컬러:
  - `primary.main`: `#E7C48E` 또는 `#DB827A`
  - `secondary.main`: `#3A2415`
  - `background.playful`: `#FBF4EC`
  - `divider`: 밝은 베이지 계열로 조정 검토
